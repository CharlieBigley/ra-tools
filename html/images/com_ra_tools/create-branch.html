How to Create a Branch for Your Changes

Creating a branch is a crucial step in collaborative development. It allows you to work on new features or bug fixes without affecting the main codebase. This guide will show you how to create and work with branches in the ra-tools project.

Why Create a Branch?

  • Isolation: Your changes won't affect the main code until they're ready
  • Organization: Each feature or fix gets its own branch
  • Collaboration: Team members can review your changes before merging
  • Safety: Easy to abandon or reset if something goes wrong

Prerequisites

  • The ra-tools repository cloned locally
  • Latest code downloaded from the main branch
  • Basic understanding of Git commands

Branch Naming Conventions

Best Practice: Use descriptive branch names that indicate the purpose of the changes:
  • feature/user-authentication - For new features
  • bugfix/fix-login-error - For bug fixes
  • hotfix/security-patch - For urgent fixes
  • docs/update-readme - For documentation updates
  • refactor/cleanup-controllers - For code refactoring

Creating a New Branch

Step 1: Navigate to your repository

cd ~/git/ra-tools

Step 2: Ensure you're on the main branch

git checkout main

This ensures you're creating your new branch from the latest stable code.

Step 3: Update your main branch

git pull origin main

This fetches and merges the latest changes from the remote repository.

Step 4: Create and switch to a new branch

Use one of these methods:

Method 1: Create and checkout in one command (recommended)

git checkout -b feature/my-new-feature

Method 2: Create then checkout separately

git branch feature/my-new-feature
git checkout feature/my-new-feature

Method 3: Using Git 2.23+ (newer syntax)

git switch -c feature/my-new-feature

Step 5: Verify you're on the new branch

git branch

The asterisk (*) should appear next to your new branch name.

Or use:

git status

This shows "On branch feature/my-new-feature" at the top.

Working with Your Branch

Step 6: View all branches

To see all local branches:

git branch

To see local and remote branches:

git branch -a

Step 7: Switch between branches

To switch to an existing branch:

git checkout branch-name

Or using newer syntax:

git switch branch-name
Tip: Use tab completion in your terminal! Type git checkout followed by the first few letters of a branch name, then press Tab to auto-complete.

Creating a Branch from a Specific Commit

Advanced: Create a branch from a specific commit

First, find the commit hash:

git log --oneline

Then create a branch from that commit:

git checkout -b my-branch-name abc1234

Replace abc1234 with the actual commit hash.

Example Workflow

Example: Creating a branch to add a new feature

# Ensure you're on main and up to date
git checkout main
git pull origin main

# Create and switch to new branch
git checkout -b feature/add-export-function

# Verify you're on the new branch
git status

# Output: On branch feature/add-export-function

Best Practices

Do's:
  • Always create a new branch for each feature or bug fix
  • Use descriptive, meaningful branch names
  • Start branches from an up-to-date main branch
  • Keep branches focused on a single purpose
  • Regularly commit your work to the branch
Don'ts:
  • Don't work directly on the main branch
  • Don't use generic names like "test" or "temp"
  • Don't include multiple unrelated changes in one branch
  • Don't keep branches alive for too long without merging

Deleting a Branch

When Done: Delete a merged branch

After your changes are merged, you can delete the local branch:

git branch -d feature/my-feature

To force delete an unmerged branch:

git branch -D feature/my-feature

What's Next?

Now that you have a branch, you can:

  • Make changes to files in the repository
  • Commit your changes incrementally
  • Push your branch to GitHub
  • Create a pull request for review

Troubleshooting

Uncommitted Changes Preventing Branch Switch

If you have uncommitted changes, Git may prevent you from switching branches. Options:

  1. Commit your changes first
  2. Stash your changes: git stash
  3. Force checkout (will lose changes): git checkout -f branch-name

Branch Already Exists

If the branch name already exists, you'll get an error. Either:

  • Choose a different name
  • Delete the existing branch first: git branch -d branch-name
  • Switch to the existing branch: git checkout branch-name
